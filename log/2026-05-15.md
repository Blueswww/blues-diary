# 开发日志 2026-05-15

## 已完成修复

### Bug 1：退出登录后缓存未清除
- **问题**：退出登录后，日记、标签、待办、纪念日等 Pinia store 的数据未清空
- **修复**：在 `mine.vue` 的 `handleLogout()` 中添加了所有 store 的 `clearCache()` 调用
- **涉及文件**：`src/pages/mine/mine.vue`

### Bug 2：删除日记功能异常
- **问题**：删除日记接口报错或未正确移除
- **修复**：重构日记 API 和 store 的删除逻辑
- **涉及文件**：`src/api/diary.ts`, `src/store/diary.ts`

### Bug 3：首页日记按日期+标签分组展示
- **问题**：同一天的多条日记各自独立成一栏，未按标签分栏
- **修复**：`groupedDiaries` 计算属性按日期分组 → 组内按标签分栏
- **涉及文件**：`src/pages/index/index.vue`
- **附加**：日历页签也完善了标签显示

### Bug 4：模板中 `&lt;` `&gt;` 显示为原文
- **问题**：日历/待办页导航箭头 `<` `>` 和"我的"页右箭头 `>` 显示为 `&lt;` `&gt;`
- **根因**：微信小程序 WXML 的 `<text>` 组件中，HTML 实体不会自动解码
- **修复**：将模板中的 `&lt;` `&gt;` 改为 Vue 表达式 `{{ '<' }}` `{{ '>' }}`
- **涉及文件**：`src/pages/calendar/calendar.vue`, `src/pages/todo/todo.vue`, `src/pages/mine/mine.vue`, `src/pages/index/index.vue`

### 预防性优化：API 层统一解码 HTML 实体
- 新增 `decodeHtml()` 在 `src/utils/decode.ts`
- 日记数据从数据库返回后立即解码，所有页面直接使用纯文本
- 移除了各页面重复的 `entryPreview()` 解码函数和 `:decode="true"`
- **涉及文件**：`src/api/diary.ts`, `src/components/DiaryCard.vue`

### WXSS 样式告警修复
- **问题**：组件 WXSS 中出现 `page{}` 标签选择器和 `scoped` 属性选择器
- **修复**：将 `page{}` 从 `uni.scss` 移到 `App.vue`；移除所有 `scoped`
- **涉及文件**：`src/uni.scss`, `src/App.vue`, 全部页面和组件
